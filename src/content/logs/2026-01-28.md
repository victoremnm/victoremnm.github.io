## Debugging Notes

The WebSocket issue on the diagnostics app was instructive. Deployment environment constraints (serverless = no persistent connections) change your architecture options significantly. The NATS JetStream HTTP API is a real escape hatch here — polling over HTTP instead of subscribing, which works fine for a diagnostics tool that doesn't need millisecond latency.

Key NATS lesson: JetStream subjects vs Core NATS subjects behave differently for pull consumers. Had to switch from a push consumer model to an explicit pull consumer to get reliable message delivery in the serverless context.

## Discoveries

- `bonkbot-org` has 17 separate git repos, not a monorepo. This matters for CLAUDE.md placement and context window management across sessions.
- ~200k files in build artifacts (`target/`, `.venv/`) were polluting VSCode file indexing. Added both to `.gitignore` and VSCode's `files.exclude`.
- dbt installation fails with system Python on macOS due to `externally-managed-environment` protection — needs a venv. Obvious in retrospect.
- Concurrent Python API calls with `asyncio.gather()` are ~2x faster than sequential (`await` in a loop): ~1.5s vs ~3s for 3 tasks. Semaphore prevents rate limiting while maintaining the speedup.
