## Learning Questions Worth Revisiting

From `web-terminal` sessions this week:

- *"Why is state being held like such, any way to be more explicit?"* — Investigating ColdFusion-style state patterns in the Rust block processor. The issue was two different types sharing the same key prefix in RocksDB.
- *"How does the discriminator be used here?"* — Program discriminators in Anchor (Solana's framework). A discriminator is an 8-byte prefix on account data that identifies which account type/instruction is being handled.
- *"Do CF holder state and CF token account state happen in sequence?"* — Tracing the migration ordering in the state machine. Yes, they're sequential — the token account state depends on holder state being current.
